3. Setting Up Google Sheets as Your Database

Your website can pull live content from a Google Sheet. This is how you update testimonials, products, FAQ, and prices without touching code.

Step A: Create The Sheet

1
Go to sheets.google.com → Click the + to create a new spreadsheet → Name it "PRML Records CMS"
2
Create 4 tabs (click the + at the bottom of the sheet): Name them exactly as: Testimonials, Products, FAQ, Services
3
Testimonials tab — add these column headers in row 1:
name | role | text | rating | approved | date
4
Products tab — add these headers:
name | category | price | description | imageurl | active
5
FAQ tab — add these headers:
question | answer | category | active

Step B: Get Your Sheet ID and API Key

1
Sheet ID: Look at your sheet's URL. It looks like: docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/edit — the bold part is your Sheet ID. Copy it.
2
Make the sheet public (read only): Click Share → change "General access" from "Restricted" to "Anyone with the link" → set to "Viewer" → Done
3
Get a Google API Key: Go to console.cloud.google.com → Create a project → Enable "Google Sheets API" → Credentials → Create Credentials → API Key → Copy the key
4
Update js/sheets-fetcher.js: Open the file in a text editor, find the lines at the top and replace the placeholder values: 
const SHEET_ID = 'PASTE_YOUR_SHEET_ID_HERE';
const API_KEY  = 'PASTE_YOUR_API_KEY_HERE';

4. Google Apps Script — Contact Form & Lead Capture

This makes your contact form send data directly to a Google Sheet AND email you every time someone submits. Completely free.

1
Go to script.google.com → Click "+ New Project"
2
Delete all the code in the editor and paste this entire block: 
function doPost(e) {
  var ss = SpreadsheetApp.openById('10hOO67uBb5rPpoFrXaW9sLm04hJSY_8yTwDb3XgUlMA');
  var data = JSON.parse(e.postData.contents);
  
  if (data.type === 'INQUIRY') {
    var sh = ss.getSheetByName('Leads');
    if (!sh) sh = ss.insertSheet('Leads');
    if (sh.getLastRow() === 0) {
      sh.appendRow(['Date','Name','Email','Phone','Business',
                    'Service','Budget','Timeline','Details','Source']);
    }
    sh.appendRow([data.ts, data.name, data.email, data.phone,
                  data.business, data.service, data.budget,
                  data.timeline, data.details, data.source]);
    
    MailApp.sendEmail({
      to: 'info@prmlrecords.com',
      subject: 'New Lead: ' + data.service,
      body: 'Name: ' + data.name + '\nEmail: ' + data.email + 
            '\nPhone: ' + data.phone + '\nService: ' + data.service +
            '\n\n' + data.details
    });
  }
  
  if (data.type === 'EMAIL') {
    var sh = ss.getSheetByName('Email List');
    if (!sh) sh = ss.insertSheet('Email List');
    sh.appendRow([data.ts, data.email]);
  }
  
  if (data.type === 'ORDER') {
    var sh = ss.getSheetByName('Orders');
    if (!sh) sh = ss.insertSheet('Orders');
    sh.appendRow([data.ts, data.items, data.total, data.amount,
                  data.depositOnly ? 'Deposit' : 'Full']);
  }
  
  return ContentService
    .createTextOutput(JSON.stringify({status: 'ok'}))
    .setMimeType(ContentService.MimeType.JSON);
}
3
Replace YOUR_SHEET_ID_HERE in line 2 with your actual Sheet ID from Step 3 above.
4
Deploy it: Click "Deploy" (top right) → New Deployment → Type: Web app → Execute as: Me → Who has access: Anyone → Deploy → Copy the URL it gives you
5
Update js/main.js: Open the file, find this line and replace the placeholder: 
const GAS_URL = 'https://script.google.com/macros/s/AKfycbzlXsUJFb36zh58jc2-N54Cmfhgh4EC478OzKehJG56qeNcZRSzFmOAU4PBHEgVdzsLyQ/exec';
6
Re-upload your site to Netlify. Now every form submission goes to your Google Sheet and emails you at hello@prmlrecords.com.
Every time you make a change to your Apps Script code, you need to create a new deployment. Go to Deploy → Manage Deployments → Edit → Save new version → Deploy. Then copy the new URL into main.js.

5. Daily Update Guide

Adding or Approving Testimonials

  1. Open your "PRML Records CMS" Google Sheet
  2. Click the "Testimonials" tab
  3. Add a new row. Fill in: name, role (like "Small Business Owner · Atlanta"), the review text, rating (1-5), and the date
  4. In the "approved" column, type TRUE (all caps) when you're ready for it to appear on the website
  5. Refresh testimonials.html — the review appears automatically
  6. To hide a review later: change approved from TRUE to FALSE

Adding or Updating Products

  1. Go to the "Products" tab
  2. Add a new row with all columns filled
  3. For the category column — use EXACTLY one of these (spelling matters): Signs, Keychains, Light Boxes, 3D Printing, Laser Cutting, Laser Engraving, Laser Edging, Sound, Film, Photography, Web Dev, Consulting
  4. For the price column: numbers only, no $ sign. Example: 35 not $35
  5. Set active to TRUE to show it, FALSE to hide it

Getting Image URLs for Products

  1. Upload your product photo to Google Drive
  2. Right-click the file → Share → change to "Anyone with the link" → Copy link
  3. Your link looks like: drive.google.com/file/d/FILE_ID/view
  4. Change it to: https://drive.google.com/uc?id=FILE_ID
  5. Paste this URL into the imageurl column in your sheet

Changing Prices

Find the product in your Products tab. Click the price cell. Type the new number (no $ sign). Press Enter. The website updates automatically when someone visits the page.

7. Stripe Payment Setup

Stripe is how customers pay for services online. It handles all the credit card processing securely.

Creating Your Stripe Account

1
Go to stripe.com → Create account → Complete business verification with your LLC info
2
In your Stripe dashboard → go to Payment Links → Create Payment Link
3
Set up a "Custom amount" payment link — this lets the cart total be passed through. Name it "PRML Records Payment"
4
Copy the Payment Link URL (looks like: buy.stripe.com/xxxx)
5
Open js/main.js → Find this line → Replace the placeholder URL: 
const stripeUrl = `https://buy.stripe.com/YOUR_PAYMENT_LINK`;

Testing Payments

  • In Stripe dashboard, toggle to "Test mode" (switch in top left)
  • Use test card: 4242 4242 4242 4242, any future date, any 3-digit CVC
  • Test purchases appear in your Stripe test dashboard
  • When ready to go live, toggle back to "Live mode" in Stripe
Important: Never share your Stripe secret key. The payment link URL is safe to include in your code. The secret key (starts with sk_) must NEVER appear in website code.

8. Troubleshooting

Testimonials Not Appearing

  • Check that the "approved" column says TRUE (all caps, no spaces)
  • Check that Google Sheets API key is correct in js/sheets-fetcher.js
  • Check that the Sheet ID is correct
  • Check that the sheet is set to "Anyone with the link can view"
  • Open browser developer tools (F12) → Console tab → look for red error messages

Images Not Showing Up

  • Make sure the Google Drive file is shared as "Anyone with the link"
  • Make sure the URL format is: https://drive.google.com/uc?id=FILE_ID — NOT the share link
  • Try opening the image URL directly in your browser to test it

Contact Form Not Sending

  • Check that GAS_URL in js/main.js is your actual deployed Apps Script URL
  • Make sure you deployed the Apps Script as a Web App with "Anyone" access
  • Try deploying a new version in Apps Script and updating the URL

Stripe Checkout Not Working

  • Check that the Stripe payment link URL is correct in js/main.js
  • Make sure you're using a Live mode payment link (not test) when you go live
  • Test with the test card: 4242 4242 4242 4242

Site Looks Different After Update

  • Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  • Try opening in a different browser or incognito window
  • If still wrong, wait 5 minutes for Netlify CDN to update
