How this queue is populated

The /prml-inbox-scan skill runs (manually or on a /schedule cron) and:

  1. Scans info@prmlrecords.com for new replies via Gmail MCP
  2. Matches each to a Sponsorship Companies row by sender email
  3. Classifies the reply against 8 contingency templates
  4. Drafts a reply in Gmail (NOT sent — just queued)
  5. Logs to Airtable Sponsorship Email Log with Replied=true + Date Sent=blank
  6. Updates Sponsorship Companies status (e.g. "Replied — Needs Triage")

This page reads Airtable rows where Email Type is set AND Date Sent is blank — that's the queue. Approve flips the row to Date Sent = today and (eventually) auto-sends via Brevo.

Current state: queue is read-only preview. Live Airtable wiring (auth proxy + write-back) ships in Phase 2 — for now, click "Open Gmail Drafts" to find the actual draft and send manually. The queue surfaces the existence of pending drafts.
